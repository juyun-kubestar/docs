配置

配置概述

作为微服务十二要素之一的配置与代码分离，配置可以作为独立的资源部署到 Kubernetes 集群里，通常环境变量用来存放与应用部署环境相关的配置信息，比如数据库连接地址、连接数最值、程序启动参数等。KubeStar™ 容器管理平台上同样将配置作为独立条目进行管理，在支持配置的创建、查看、修改以及删除等基本操作的同时，还支持在应用发布的时候以配置卷的形式使用配置，以及将配置注入应用环境变量。

创建配置

  • 点击左上方的导航按钮，点击创建配置

  • 创建配置主要分为两部分

    1. 填写配置基本信息

      • 配置名称：为配置起一个简洁明了的名称，便于用户浏览和搜索

      • 命名空间：配置所属的命名空间

      • 位置：配置要发布到的云服务商/地区/集群

    2. 填写配置项信息

      点击”创建“增加一条新的配置条目。每个配置条目由 Key 和 Value 进行管理和保存。例如对于一个名为 default.yml 的配置文件，其内容为：

      monitoring: true
metrics: true
vendors:
  - aws
prometheus:
host: abde20d3df16411e98fee0e393284b1d-44833454.ap-northeast-1.elb.amazonaws.com

      在”新增配置项“的时候可以使用文件名 default.yml 作为配置条目的 Key，文件内容作为配置条目的 Value，当配置条目填写完毕以后，点击”提交“将该条目添加到该配置中，最后点击”提交“即可在 default 命名空间下创建该名为”myconfig“的配置

使用配置

配置成功创建后，即可被应用所使用，共有两种方式：

  • 在通过 KubeStar™ 创建应用的时候，可以在容器组配置-新增容器的时候使用配置

  • 将配置注入应用环境变量
