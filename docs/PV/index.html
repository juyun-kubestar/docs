存储卷

存储卷概述

KubeStar™ 容器管理平台支持为应用提供多元化的数据存储服务，包括手动指定卷类型，以及自动创建所需要的卷。类似配置管理，在支持存储卷的创建、查看以及删除等基本操作的同时，还支持在应用发布的时候将卷挂载到用户指定的路径，同时提供多种挂载策略供用户选择使用。

创建存储卷申请

  • 点击左上方的导航按钮，点击创建存储卷申请

  • 填写存储卷申请配置信息

    • 名称：为该申请起一个简洁明了的名称，便于用户浏览和搜索

    • 命名空间：配置所属的命名空间

    • 位置：配置要发布到的云服务商/地区/集群

    • 访问模式：表示该存储卷的读写访问模式

      • ReadWriteOnce — 可以被单个节点以读/写模式挂载

      • ReadOnlyMany — 可以被多个节点以只读模式挂载

      • ReadWriteMany — 可以被多个节点以读/写模式挂载

    • 容量：该存储卷的大小

    • 卷类型：该卷的挂载方式

      • Filesystem：格式化

      • Block：不执行格式化，保留原始块设备

使用存储卷申请

存储卷申请创建成功后，有两种方式创建对应的存储卷：

  • 手动：运维人员根据机房资源情况按照存储卷申请的配置创建存储卷

  • 自动：公有云根据存储卷申请的配置自动创建存储卷

  • 应用挂载存储卷

    • 在通过 KubeStar™ 创建应用的时候，可以在容器组配置-新增容器的时候使用存储卷
