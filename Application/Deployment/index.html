无状态应用

无状态应用管理概述

发布无状态应用

发布流程：

  • 点击左上方的导航按钮，在发布通道中点击发布应用

  • 选择无状态应用进行应用配置信息填写

    • 应用配置信息主要分为三部分：

      1. 基本配置：应用的基本配置

        无状态应用表示该应用可以无限水平扩展，实例间可以相互替代，比如 Nginx

        • 应用名称：无状态应用的名称

        • 位置：应用要发布到的云服务商/地区/集群

        • 命名空间：应用所属的命名空间

        • 副本数量：副本数量表示用户期望的实例数量

        • 停止等待时间：表示平台留给应用退出的执行时间，超过此时间的应用会被强制退出

      2. 容器组配置：

        该应用所包含的容器组的相关配置，表示为该应用添加的多个容器，一般情况下第一个容器为主业务容器，其它容器都是辅助容器。每个容器的配置是一样的，需要填写基本信息，包括容器名称、选择镜像、CPU、内存及开放端口等基本信息，以及下面的高级配置、存储卷、环境变量、生命周期及健康检查

        • 配置：点击配置下的创建按钮，以选择之前创建好的配置名称，并指定配置存放的绝对路径和每个配置项的具体文件路径

        • 存储卷：点击存储卷下的创建按钮，可以选择创建好的存储卷申请，并指定卷挂载的绝对路径

        • 环境变量：点击环境变量可输入或选择环境变量：

          • 添加环境变量：分别填入环境变量的 Key 和 Value。如：Name：KUBECONFIG，Value：/home/.kube/config

          • 添加配置文件/密钥引用：先填入环境变量的 Key，值是从配置中读取，然后选择该配置的名称，以及配置中对应的键名。

        • 生命周期:点击生命周期可选择容器启动命令、容器启动后及停止前处理的命令，处理方式包括命令行方式、HTTP 请求方式、TCP 请求方式

        • 注意：这里启动命令项的运行命令目前支持以 , 分隔的命令输入，启动后处理和停止前处理项的执行命令目前支持单一命令的输入（不含参数）

        • 健康检查

          • 应用存活探针：检测应用健康状态，动作可以是脚本、HTTP 地址或 Socket 端口，通常用来检测应用负载，如果多次执行失败应用会被重启和应用就绪探针”
          • 应用就绪探针：检测应用是否就绪，动作可以是脚本、HTTP 地址或 Socket 端口，通常用来检测应用是否可以接收请求，如果多次执行失败请求不会转发到该应用

      3. 服务配置：该应用暴露的服务访问端点

      点击添加服务可以为该应用添加多个访问端点，其中要填写服务名称，添加多个开放端口、选择服务类型。服务类型包含：

      • 内部服务（ClusterIP）：存在 Cluster IP 地址, 只能在 kubernetes 集群内部访问, Kubernetes 集群外部访问不可达

      • 内部服务（Headless）：不存在 Cluster IP 地址, 只能在 Kubernetes 集群内部访问, 只能使用 Kubernetes 短域名访问,通常使用于同一个 Kubernetes 集群内服务之间的相互调用

      • 外部服务（NodePort）：在集群外使用 Kubernetes 任何 work 节点的 IP+ NodePort 端口就能访问，通常私有云在公司内网会支持这种方式,无法满足公网的访问需求

      • 外部服务（LoadBalancer）：作为 NodePort 的补充, 可以用于 Kubernetes 集群外部访问或公网访问,单通常需要开发 LB, LB 会返回一个访问 host, 例如 ALB,ELB, 现在 kubestar 支持 ALB

  • 提交，服务配置完成后会在页面显示，点击提交即可完成无状态应用发布
